# Fiber_DNS_SBT: inertial slender fibers in homogeneous isotropic turbulence

An MPI-parallel Fortran framework coupling pseudo-spectral direct numerical simulation (DNS) of homogeneous isotropic turbulence with inertial slender-body theory (SBT) for finite-Reynolds-number fiber dynamics.

## Overview

`Fiber_DNS_SBT` is designed to simulate the coupled evolution of an incompressible turbulent flow and one or more slender fibers in a triply periodic domain.

The framework combines:

- a three-dimensional pseudo-spectral DNS solver for the incompressible Navier-Stokes equations;
- stochastic large-scale forcing for statistically stationary homogeneous isotropic turbulence;
- an inertial slender-body theory for the force per unit length along each fiber;
- interpolation of the fluid velocity onto points along each fiber axis;
- periodic-image corrections based on Ewald summation; and
- iterative two-way coupling between the fiber dynamics and the turbulent carrier flow.

At each coupling step:

1. a current estimate of the force distribution along every fiber is spread to the fluid;
2. the resulting fiber body force is transformed to Fourier space;
3. the pseudo-spectral solver updates the fluid disturbance;
4. the non-singular fluid velocity is reconstructed and interpolated along each fiber axis;
5. the inertial SBT equation is solved for an updated force distribution; and
6. the fiber translational and rotational motion is updated.

This approach captures fiber-scale hydrodynamic forcing without directly resolving the fiber thickness on the DNS grid.

## Features

- MPI-parallel Fortran implementation
- triply periodic pseudo-spectral DNS
- homogeneous isotropic turbulence with stochastic forcing
- second-order Runge-Kutta time integration
- phase-shift and spectral-truncation options for dealiasing
- inertial SBT for finite-Reynolds-number slender fibers
- iterative two-way fiber-fluid coupling
- distributed three-dimensional FFTs
- Hermitian-symmetry reduction in Fourier space
- interpolation of fluid velocity along the fiber axis
- periodic-image correction using Ewald summation
- prescribed or dynamically evolving fiber motion
- optional finite fiber inertia
- support for gravity and externally imposed forces and torques
- output of turbulence statistics and fiber-dynamics quantities

## Scientific applications

The framework can be used to investigate:

- orientation and rotation-rate statistics of slender fibers in turbulence;
- settling of inertial fibers in homogeneous isotropic turbulence;
- preferential alignment with strain-rate, vorticity, and gravity directions;
- modification of turbulence by fiber forcing;
- one-way and two-way fiber-fluid coupling;
- spectral transfer between fibers and turbulence;
- effects of fiber length relative to the Kolmogorov scale;
- effects of fiber aspect ratio, density, and inertia; and
- transient and statistically stationary particle-flow dynamics.

## Governing methodology

The carrier flow satisfies the incompressible Navier-Stokes equations in a periodic domain. Spatial derivatives are evaluated spectrally, while nonlinear products are formed in physical space. The fiber force is represented by a line distribution along each fiber axis and is coupled to the fluid as a body force.

The force distribution is obtained from an inertial SBT integral equation using the non-singular velocity disturbance evaluated along the fiber. The singular Stokes disturbance generated by the fiber itself is removed in Fourier space, and the periodic Stokes contribution from image fibers is restored using an Ewald-summation correction.

## Public-repository status

The public repository currently contains the theoretical and numerical documentation, a representative input file, example results, and build/run templates.

The main Fortran source code is maintained in a private repository while the associated research is ongoing. The build and execution instructions below apply when the source files are available.

## Build requirements

Typical requirements include:

- a Fortran compiler with MPI support;
- an MPI implementation;
- BLAS and LAPACK;
- ScaLAPACK;
- Make;
- Python or MATLAB for optional post-processing.

The supplied Makefile is configured for an Ubuntu or WSL environment using:

```text
mpifort
-lscalapack-openmpi
-llapack
-lblas
```

Library names may need to be modified for a different system or HPC software stack.

## Build

When the source files are available in the working directory, compile using:

```bash
make -f scripts/Makefile
```

This produces:

```text
fiber_in_HIT_prog
```

To remove object files, module files, and the executable:

```bash
make -f scripts/Makefile clean
```

## Run

The MPI process count must be consistent with the decomposition specified in the input file:

$$N_{\mathrm{proc}}=2^{d_{\mathrm{proc}}}=2^{d_x+d_y+d_z}.$$

For the supplied sample input, $d_{\mathrm{proc}}=6$, so the required process count is $64$.

Example:

```bash
mpirun -np 64 ./fiber_in_HIT_prog
```

Alternatively:

```bash
bash scripts/run_example.sh
```

The executable reads the Fortran namelist from `input_file`. Depending on how the private source tree is organized, copy or link `input/input_file` into the execution directory before launching the calculation.

## Input configuration

The sample namelist controls:

- DNS resolution and MPI decomposition;
- time integration and output intervals;
- periodic-domain dimensions;
- nonlinear-term truncation and dealiasing;
- fluid and fiber properties;
- number of fibers and axial SBT resolution;
- initial fiber orientation and motion;
- gravity, external force, and external torque;
- interpolation order;
- stochastic turbulence forcing; and
- the output directory.

See [`docs/input_parameters.md`](docs/input_parameters.md) for a parameter-by-parameter reference.

## Example results

The `examples/` directory contains representative results for inertial fibers settling in homogeneous isotropic turbulence, including:

- a simulation video;
- turbulent kinetic energy;
- dissipation rate; and
- root-mean-square turbulent velocity fluctuations.

The example case uses a cubic periodic domain and introduces fibers after the turbulent flow has reached statistical stationarity.

## Numerical verification

New simulations should be checked for convergence with respect to:

- DNS resolution;
- time-step size;
- fiber-axis resolution;
- interpolation order;
- Ewald cutoff radius;
- dealiasing strategy;
- iterative fiber-fluid coupling tolerance; and
- MPI decomposition.

The turbulence resolution should also be assessed using quantities such as $k_{\max}\eta$.
